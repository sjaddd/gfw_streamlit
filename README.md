# GFW Vessel Behaviour Classifier: Streamlit App

Interactive demonstration of fishing vessel behaviour classification using AIS tracking data from Global Fishing Watch.

## Prerequisites

- Python 3.9+
- Git LFS (required to clone model and data files)

## Setup

```bash
# 1. Create and activate a virtual environment
python -m venv .venv
.venv\Scripts\activate        # Windows
source .venv/bin/activate     # Mac/Linux

# 2. Install dependencies
pip install -r requirements.txt
```

## Running the App

```bash
streamlit run app.py
```

The app will open automatically in your browser at `http://localhost:8501`

## App Tabs

| Tab | Description |
|-----|-------------|
| ▶ Replay vessel track | Animates real AIS vessel tracks from the test set with live model predictions, running F1 scores, and ground truth comparison |
| 🐟 How vessels fish | Synthetic animation demonstrating the characteristic movement patterns of each gear type |
| 🌍 Ocean Region Predictor | Interactive predictor for fishing activity by ocean region using a separate Random Forest model |

## Folder Structure

```
├── app.py                    # Main Streamlit application
├── replay_component.html     # Vessel track animation component
├── vessel_patterns.html      # Gear type pattern animation component
├── requirements.txt          # Python dependencies
├── models/
│   ├── knn_gear.pkl          # k-Nearest Neighbours model
│   ├── lr_gear.pkl           # Logistic Regression model
│   ├── rf_gear_n20.pkl       # Random Forest model (20 estimators)
│   ├── nb_gear.pkl           # Naive Bayes model
│   ├── scaler_gear.pkl       # StandardScaler fitted on Q2 features
│   ├── test_trip_ids.pkl     # Held-out test set trip IDs
│   └── model_meta.json       # Feature names and model metadata
└── data/
    └── gfw_features.parquet  # Engineered feature dataset
```

## Notes

- **Models**: These were generated by the notebook's "Models Export" section. 
- **Random Forest model**: The app uses a 20-estimator version (`rf_gear_n20.pkl`) rather than the 100-estimator model in the notebook, due to file size constraints (678MB vs 135MB). Performance difference is negligible (F1: 0.866 vs 0.870).
- **Models are Q2 variants**: All models were trained with gear type dummy variables as additional features alongside the kinematic and temporal features.
- **Test set only**: The app displays trips from the notebook's held-out test set only, ensuring no trip shown was used during model training. Additionally, only trips with a combination of fishing and non-fishing activity is present are shown ("interesting" trips)
- **Live deployment**: The app is also deployed at: https://your-app-name.streamlit.app
